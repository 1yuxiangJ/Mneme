## 关键决策 / 发现(本次会话的关键转折)

### 1. 项目方向从 "Letta Career Hub" 改成 "Memory Service for Claude Code"

**之前**:Career Hub MVP(3 张表 + 1 个 Career Agent + ReAct + 等)
**现在**:Memory-as-a-Service 通过 MCP 接 Claude Code

**理由**:
- Dogfooding 强(用户每天写代码都用 Claude Code)
- Narrative 强("给 Claude Code 装上 Letta 式记忆")
- Demo 直观(打开 Claude Code 就能演示)
- 跟实习零重合

### 2. 必须做 Sleep Agent("做梦机制"),不只是 memory backend

**理由**:如果只做 memory backend,退化为"LLM 包装的 CRUD"——跟用户拒绝的 ViMax 是一个套路。

**Sleep Agent 让项目算"真 agent"的关键**:它是完全自主、unbounded、ReAct loop in background,无外部触发。

**论文背书**:Letta 团队 2024 年提出 "sleep-time compute"(详见 PLAN.md §3.2)。

### 3. Memory scope 只做 global,不做 project-scoped

**用户原话戳到根**:"如果写入记忆还要区分 project,那写入记忆的动机是什么?回到原来对话窗口里就完全可以啊"

**最终边界**:
- Project 内的事:**Claude Code 自己**(CLAUDE.md + auto memory)
- 跨 project 的"关于用户这个人":**dream 服务**

**LLM 通过 system prompt 学会判断该不该写 dream**。

### 4. 语言从 Java 改 Python

**理由**:Letta / MCP / agent 生态全是 Python。Java 是 outlier,简历叙事用 Python 反而不用解释。
**简历策略**:双线投(Java backend 实习对口 + AI agent 项目 Python 差异化)。

### 5. ReAct loop 用 LangGraph,不手写

**理由**:校招学生说"自研 ReAct + heartbeat" 可信度低,LangGraph 是行业惯例,会用主流框架本身就是能力。
**项目灵魂在 Sleep Agent 算法 + 并发安全设计 + memory schema**,不在 agent loop 自研。

### 6. LLM 全程 DeepSeek(改自最初"Sleep 用 Claude")

**理由**:用户有 DeepSeek 额度,学生项目省钱优先。

### 7. 开源代码可复用 70%,自己写 30%

- ✅ 直接 pip install:FastAPI / SQLAlchemy / pgvector / mcp / langgraph / langchain-openai / apscheduler / pytest
- ✅ Example 套用:MCP server 骨架、LangGraph ReAct、pgvector retriever
- ❌ 自己写:Memory schema、Tool handlers、**Sleep Agent 完整**、并发 staging swap、System prompts

**那 30% 正好是简历项目灵魂,GitHub 没有现成的 fork 来的**。
